# SafePass Client Apps — Handoff Package

Two operator-facing applications to build against SafePass's existing backend:
**Visitor Management** and **Mapping**. Suggested reading order:

1. `1-overview.md` — one-page overview of the two apps.
2. `2-requirements-brief.md` — the detailed functional brief: screens, user journeys,
   platform constraints, and what is in and out of scope. The primary document for
   scoping and pricing.
3. `3-api-spec.html` — browsable API reference for exactly the endpoints these two
   apps use (open in a browser). `3-api-spec.yaml` is the same specification for
   tooling import (Swagger, Postman, code generators).
4. `4-design-reference.html` — visual design reference (open in a browser).
5. `5-questions-answered.md` — answers to the questions raised so far.
